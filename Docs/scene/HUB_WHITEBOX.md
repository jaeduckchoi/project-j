# 허브 화이트박스 기준

이 문서는 `Hub.unity`의 맵 크기, 화면 연출 목표, 이동 경계, 현재 프로젝트 정합성 점검 결과를 정리한 정본 문서다.  
허브의 최종 배치 정본은 씬 직렬화 값이고, 논리 타일 계약은 `HubRoomLayout.cs`를 따른다.

## 1. 화면 기준 정본

- 화면 연출 목표: `1920x1080`
- 목표 비율: `16:9`
- 화면 수: `1화면`
- 월드 기준: `32x18`
- 해석: 허브 전체가 한 화면 안에 들어오는 구성을 목표로 둔다.

허브는 탐험 맵과 달리 준비, 조리, 테이블, 출구가 같은 화면 안에서 동시에 읽혀야 한다.  
`Close`에서는 준비와 출구 동선이, `Open`에서는 조리와 서빙 동선이 한 화면 안에서 곧바로 이해되는 구성을 기준으로 삼는다.

## 2. 현재 프로젝트 정합성 점검

### 판정

`정합`

### 근거

- `Hub.unity`의 카메라는 `orthographic size 9`를 사용한다.
- `Hub.unity`의 `CameraBounds`는 `32x18`이다.
- `Hub.unity`의 UI 기준 해상도는 `1920x1080`이다.
- `HubRoomLayout.cs`는 허브를 `32x18`, `16:9`, 한 화면 계약으로 설명한다.

즉 현재 허브는 `1920x1080` 1화면 목표와 맞물리도록 설계되어 있다.

## 3. 이동 경계 기준

- `CameraBounds`: `32x18`
- `HubMovementBounds`: `31.166668x17.166668`

실제 이동 경계는 전체 허브 크기보다 살짝 작다.  
하단 전경선과 벽 두께 때문에 플레이어 이동 가능 영역만 안쪽으로 줄여 둔 상태다.

## 4. 타일 원본 기준 해석 메모

- 이 문서의 `1920x1080`은 `화면 기준 px`다.
- 현재 프로젝트의 타일 import 계약은 `64px` 타일 기준을 유지한다.
- 타일 원본 기준 px와 화면 기준 px를 같은 의미로 취급하지 않는다.
- 타일 원본 기준으로 exact matching하려면 `60px` 타일 계약 전환이 필요하다. 이번 범위에는 포함하지 않는다.
