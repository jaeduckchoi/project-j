# 커밋 메시지 생성 규칙

이 문서는 이 저장소의 Git 커밋 메시지 규칙 정본이다.
커밋 메시지를 생성할 때는 다른 문서 요약이나 일반 Git 관례보다 이 문서를 우선 적용한다.

## 1. 기본 원칙

- 모든 Git 커밋 메시지는 한국어 한 줄로 작성한다.
- 형식은 `타입 : 내용`만 허용한다.
- 전체 한 줄은 50자 이내로 유지한다.
- `타입`은 아래 표의 값만 사용한다.
- 여러 변경이 섞여 있으면 가장 대표적인 변경 1가지만 잡아 제목을 만든다.

## 2. 절대 출력 규칙

- 최종 출력은 커밋 메시지 한 줄만 남긴다.
- 머리말, 설명 문장, 불릿 목록, changelog 요약, 코드블록은 금지한다.
- 여러 후보를 나열하지 않는다.
- `추천:`, `예시:`, `커밋 메시지:` 같은 안내 문구를 앞에 붙이지 않는다.
- 백틱, 따옴표, 괄호로 커밋 메시지를 감싸지 않는다.
- 영어 문장형 제목, `docs: ...` 같은 콜론 형식, 자동 생성 diff 요약은 금지한다.
- 제목 끝에 마침표를 붙이지 않는다.
- 파일 경로, 코드 식별자, 브랜치명처럼 번역하면 안 되는 고유 명칭 외에는 영문을 쓰지 않는다.
- 50자를 넘기면 실패로 간주하고 더 짧게 다시 쓴다.

## 3. 내용 작성 규칙

- `내용`은 무엇이 달라졌는지 바로 드러나게 쓴다.
- 작업 과정 설명보다 결과나 변경 대상이 보이도록 쓴다.
- `여러 수정`, `기타 정리`, `작업 반영`처럼 의미가 흐린 표현은 피한다.
- 문서 변경이면 문서 영역이나 정리 대상이 드러나게 쓴다.
- 버그 수정이면 증상이나 잘못된 동작이 드러나게 쓴다.
- 파일명 나열식보다 기능, 화면, 흐름 중심 표현을 우선한다.

## 4. 타입 표

| 타입 | 의미 |
| --- | --- |
| `feat` | 새로운 기능 추가 |
| `update` | 기존 기능 조정 |
| `fix` | 버그 수정 |
| `docs` | 문서 또는 주석 수정 |
| `design` | UI 또는 시각 표현 변경 |
| `style` | 동작 변화 없는 형식 수정 |
| `rename` | 파일 또는 식별자 이름 변경 |
| `delete` | 불필요한 파일 또는 코드 삭제 |
| `refactor` | 동작 유지 전제의 구조 개선 |
| `test` | 테스트 추가 또는 개선 |
| `chore` | 빌드, 설정, import 정리 등 유지보수 |

## 5. 형식

```text
타입 : 내용
```

## 6. 예시

- `feat : 허브 팝업 흐름 정리`
- `fix : 폐광 포털 잠금 안내 수정`
- `docs : .aiassistant 문서 체계 정리`
- `refactor : 식당 결과 팝업 분기 구조 개선`

## 7. 금지 예시

- `docs: Add agent workflow and source-of-truth rules to project guide`
- `커밋 메시지: docs : 문서 정리`
- `추천 1) docs : 문서 정리`
- 여러 줄 본문이 붙은 커밋 메시지
- 영문 bullet 요약이 붙은 커밋 메시지
- `타입 : 내용` 앞뒤에 설명 문장이 추가된 출력

## 8. 생성 전 자가 점검

1. `타입`이 허용된 표 안에 있는지 확인한다.
2. 출력이 정말 한 줄만 남았는지 확인한다.
3. `타입 : 내용` 형식을 정확히 지켰는지 확인한다.
4. 설명, 후보, 따옴표, 백틱이 섞이지 않았는지 확인한다.
5. 50자 이내인지 확인하고 넘기면 더 줄인다.

## 9. 커밋 메시지 생성 입력 템플릿

커밋 메시지를 생성할 때는 아래 지시를 그대로 우선 적용한다.

```text
이 저장소의 Git 커밋 메시지 규칙만 따른다.
최종 출력은 커밋 메시지 한 줄만 남긴다.
설명, 머리말, 후보 나열, 불릿, 코드블록, 따옴표 출력 금지.
형식은 타입 : 내용만 사용한다.
타입은 허용된 표에서 하나만 고른다.
내용은 핵심 변경 1가지만 짧게 요약한다.
전체 50자 이내 한국어로 작성한다.
```

## 10. 템플릿 운영 규칙

- 이 파일은 로컬 `commit.template`의 정본이다.
- 다른 문서에는 이 규칙을 중복 복사하지 말고 이 파일만 참조한다.
- 커밋 규칙을 바꾸면 엔트리 파일과 관련 project 문서의 링크만 함께 확인한다.
