# 식당 영업과 성장 기획

## 역할

이 문서는 허브가 어떤 공간으로 읽혀야 하는지와, 식당 운영이 성장 축과 어떻게 이어지는지를 설명하는 정본이다.

## 이 문서를 읽는 시점

- 허브 공간의 의미, 오브젝트 역할, 성장 방향을 정리할 때
- `scene:room`과 장기 성장 축을 검토할 때
- 정확한 `Open/Close` 활성 규칙과 주문 수치는 [GAMEPLAY_CORE_LOOP.md](GAMEPLAY_CORE_LOOP.md)를 따른다.

## 정본 범위

### 허브의 역할

허브는 준비, 영업, 조리, 탐험 출발, 성장의 중심 공간이다.

- `Close`: 준비와 탐험 출발
- `Open`: 주문 대응과 조리 압박

### 허브 오브젝트 역할

| 오브젝트 | 현재 역할 | 비고 |
| --- | --- | --- |
| 냉장고 | 보유 재료 확인 | 현재 단계에서는 보관과 인출보다 확인에 집중한다. |
| 레시피 | 해금 요리, 필요 재료, 조리기구 확인 | 메뉴 선정의 근거를 제공한다. |
| 메뉴판 | 오늘의 메뉴 3칸 선택 | 정확한 메뉴 게이트와 화이트리스트는 [GAMEPLAY_CORE_LOOP.md](GAMEPLAY_CORE_LOOP.md) 정본이다. |
| 포스기 | 후속 기능 자리 확보 | 매출, 리뷰, 업그레이드 상세는 후속이다. |
| PassCounter | 결과물 적재와 회수 | 허브 코어에서는 작업대 의미가 우선이다. |
| CookingUtensils | 재료 선택과 조리 시작 | 도마, 냄비, 후라이팬, 튀김기로 구성된다. |

### `scene:room`과 성장 축

`scene:room`은 보조 성장 공간이다.

- 도구 작업대: 수리와 업그레이드
- 아이템 컬렉션: 후속
- 다이어리: 후속

영업 중 `scene:room` 접근 차단은 허브 코어 활성 규칙의 일부이며, 정확한 상태 전환은 [GAMEPLAY_CORE_LOOP.md](GAMEPLAY_CORE_LOOP.md)에서 관리한다.

### 인벤토리와 성장

- 탐험으로 재료를 모으고 허브에서 소모한다.
- 골드는 영업 보상과 성장 비용에 쓰인다.
- 이동 편의, 탐험 효율, 도구 성장, 허브 편의 확장은 장기 성장 축이다.

## 후속 분리 후보

- 고객 동선 및 좌석 확장 규칙
- 포스기 후속 기획
- `scene:room` 상세 기획
- 팝업, 시간 정지 규칙
- 탐험 리스크, 보상 표

공간 크기와 화면 기준은 [HUB_WHITEBOX.md](../Scene/HUB_WHITEBOX.md), [BEACH_WHITEBOX.md](../Scene/BEACH_WHITEBOX.md)를 따른다.
