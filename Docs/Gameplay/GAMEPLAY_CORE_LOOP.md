# 게임플레이 코어 루프

## 역할

이 문서는 허브 코어 루프의 운영 규칙, 상태 수치, 조리 계약의 정본이다.

## 이 문서를 읽는 시점

- `Open/Close`, 메뉴 게이트, 주문 타이밍, 조리 흐름을 바꿀 때
- 허브 코어 루프와 연결된 UI, 씬, 상호작용 수치를 검토할 때
- 함께 읽을 문서: [GAMEPLAY_RESTAURANT_AND_GROWTH.md](GAMEPLAY_RESTAURANT_AND_GROWTH.md), [HUB_WHITEBOX.md](../Scene/HUB_WHITEBOX.md)

## 정본 범위

### 최상위 흐름

1. `Hub (Close)`에서 냉장고, 레시피, 메뉴판을 확인한다.
2. 필요한 재료가 부족하면 탐험으로 나간다.
3. 오늘의 메뉴 3종을 정한다.
4. `OPEN`으로 영업을 시작한다.
5. 손님 주문을 확인한다.
6. 해당 `CookingUtensils`에서 재료를 선택하고 조리를 시작한다.
7. 조리 완료 결과물을 회수한다.
8. 결과물을 `PassCounter`에 적재하거나 바로 든다.
9. 주문한 테이블에 서빙한다.
10. 영업 종료 후 다시 `Close`로 돌아간다.

### 허브 상태 규칙

| 구분 | `Hub (Close)` | `Hub (Open)` |
| --- | --- | --- |
| 냉장고 | 활성 | 활성 |
| 레시피 | 활성 | 활성 |
| 메뉴판 | 활성 | 비활성 |
| PassCounter | 비활성 | 활성 |
| CookingUtensils | 비활성 | 활성 |
| `scene:room` 이동 | 활성 | 차단 |
| `scene:beach` 이동 | 활성 | 차단 |
| `OPEN` UI | 시작 가능 | 영업 중 표시 |

- 오늘의 메뉴 3칸이 모두 채워져야 `OPEN`이 가능하다.
- 메뉴가 비어 있으면 안내만 표시하고 영업을 시작하지 않는다.

### 손님 주문과 서빙

- 손님은 착석 후 2초 뒤 오늘의 메뉴 중 하나를 주문한다.
- 주문은 아이콘 말풍선으로 표시된다.
- 기본 인내심은 10초다.
- 주문과 일치하는 완성 요리를 손에 든 상태에서 해당 테이블과 상호작용하면 주문이 완료된다.

`10초`는 현재 프로토타입 화이트박스 검증값이다. 확정 수치라기보다 허브 동선과 조리 압박을 재는 기준값으로 본다.

### 조리 루프

#### 냉장고

- 냉장고는 보유 재료 확인 전용이다.
- 기본 재료 `김치`, `밥`, `고춧가루`, `밀가루`는 무한 사용이다.
- 기타 재료는 현재 보유 수량만큼만 사용할 수 있다.

#### CookingUtensils

- 플레이어는 기구 패널에서 재료를 선택한다.
- 선택한 재료 조합이 유효한 레시피와 일치하면 조리를 시작할 수 있다.
- 조리 시작 시 재료가 소모된다.
- 도마는 수동 게이지, 냄비, 후라이팬, 튀김기는 자동 게이지다.
- 잘못된 조합은 실패 요리를 만들지 않고 조리가 시작되지 않는다.

#### PassCounter

- 조리 완료 결과물을 잠시 올려둘 수 있다.
- 현재 슬롯 수는 6칸이다.
- 올린 결과물은 다시 집어 손에 들 수 있다.

### 프로토타입 요리 처리 규칙

- 현재 테스트 레시피 3종은 모두 단일 조리 단계로 처리한다.
- 기구 조리 완료 결과물은 곧바로 서빙 가능한 완성 요리다.
- 예전 `Bundle -> Finalize` 흐름은 허브 코어 정본 경로에서 제외한다.

### 테스트 레시피

현재 허브 코어 런타임의 메뉴 후보 화이트리스트는 아래 3종으로 고정한다. 탐험 확장용 generated 레시피와 자원 데이터는 유지하지만 이번 정합화 범위에서는 허브 메뉴 후보에 노출하지 않는다.

| 레시피명 | 조리기구 | 재료 1 | 재료 2 |
| --- | --- | --- | --- |
| 김치볶음밥 | 후라이팬 | 김치 | 밥 |
| 김치찌개 | 냄비 | 김치 | 고춧가루 |
| 김치전 | 후라이팬 | 김치 | 밀가루 |

## 현재 제외 범위

- 총 방문 손님 수 제한
- 포스기 상세 기능
- 아르바이트생 자동 서빙
- 좌석 확장 규칙과 2인 1조 연출
