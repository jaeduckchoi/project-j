# 종구의 식당 게임 기획 개요

이 문서는 현재 프로토타입의 게임 기획 개요 정본이다.  
세부 규칙은 같은 폴더의 하위 문서에서 다루고, 씬 직렬화와 코드 결합 지점은 `Docs/project/SOURCE_OF_TRUTH.md`를 따른다.

## 1. 게임 개요

종구의 식당은 작은 섬을 배경으로, 탐험과 식당 운영을 오가는 2D 탑다운 운영형 게임이다.

- 플레이어는 번아웃을 겪은 황종구가 되어 외딴 섬에서 자신만의 식당을 연다.
- 탐험 루프에서 식재료를 모으고, 운영 루프에서 오늘의 메뉴를 정하고 손님 주문을 조리하고 서빙한다.
- 재료 수급과 영업 보상은 서로를 강화하는 구조로 이어진다.

현재 허브 코어의 조리 흐름은 아래를 기준으로 한다.

`냉장고 확인 -> CookingUtensils에서 재료 선택 -> 조리 시작 -> 결과물 회수 -> PassCounter 적재/회수 -> 서빙`

## 2. 플레이 루프

1. 식당 허브에서 보유 재료를 확인한다.
2. 레시피와 메뉴판으로 오늘의 메뉴 3종을 정한다.
3. 필요한 재료가 부족하면 탐험 지역으로 나가 수급한다.
4. 허브로 돌아와 `OPEN`으로 영업을 시작한다.
5. 손님 주문을 확인한다.
6. 해당 조리기구에서 재료를 선택하고 조리를 시작한다.
7. 완성된 요리를 `PassCounter`에 올리거나 바로 회수한다.
8. 주문한 손님에게 서빙하고 보상을 얻는다.
9. 영업 종료 후 다시 준비, 탐험, 성장을 반복한다.

## 3. 기본 스토리와 인트로 의도

- 도시 대기업 직원이었던 황종구는 반복되는 업무와 번아웃 속에서 음식점 창업을 꿈꾼다.
- 자신만의 가게를 차릴 장소를 찾던 종구는 육지와 떨어진 작은 섬에서 새로운 시작을 결심한다.
- 플레이어는 식당과 주변 환경을 처음 둘러보며, 식당 운영과 섬 탐험이 같은 삶의 루프로 연결된다는 구조를 익힌다.

## 4. scene:hub / 종구의 식당

허브는 `Close`와 `Open` 두 상태를 가진다.

### 4-1. Close

- 냉장고: 활성
- 레시피: 활성
- 메뉴판: 활성
- 포스기: 후속 기획 위치만 유지
- PassCounter: 비활성
- CookingUtensils(도마, 냄비, 후라이팬, 튀김기): 비활성
- `scene:room` 이동: 활성
- `scene:beach` 이동: 활성

Close는 준비와 탐험 출발을 담당하는 페이즈다.

### 4-2. Open

- 냉장고: 활성
- 레시피: 활성
- 메뉴판: 비활성
- 포스기: 비활성
- PassCounter: 활성
- CookingUtensils: 활성
- `scene:room` 이동: 차단
- `scene:beach` 이동: 차단

Open은 운영과 조리에 집중하는 페이즈다.

### 4-3. OPEN 시작 조건

- 오늘의 메뉴 3칸이 모두 채워져 있어야 한다.
- 미설정 상태면 영업 시작 대신 메뉴 확인 안내만 표시한다.

## 5. 허브 상호작용 오브젝트

### 냉장고

- 현재 조리에 사용할 수 있는 재료 확인용 팝업이다.
- 기본 재료 `김치`, `밥`, `고춧가루`, `밀가루`는 상시 사용 가능하다.
- 그 외 재료는 탐험으로 확보한다.

### 레시피

- 해금한 요리 목록과 필요 재료, 사용 조리기구, 판매 가격을 확인한다.

### 메뉴판

- 오늘의 메뉴 3칸을 직접 채운다.

### PassCounter

- 조리 완료품을 잠시 올려두고 다시 집을 수 있는 작업대다.
- 현재 프로토타입은 6칸을 사용한다.
- 허브 코어에서는 결과물 적재와 회수에 집중한다.

### CookingUtensils

- 도마, 냄비, 후라이팬, 튀김기로 구성된다.
- 각 기구에서 재료를 선택하고 유효한 조합일 때만 조리가 시작된다.
- 조리 시작 시 재료가 소모된다.
- 도마는 수동 게이지, 나머지는 자동 게이지다.
- 잘못된 조합이면 실패 요리를 만들지 않고 아무 일도 일어나지 않는다.

## 6. 손님 주문과 서빙

- 손님은 착석 후 2초 뒤 오늘의 메뉴 중 하나를 주문한다.
- 주문 아이콘이 머리 위에 표시된다.
- 기본 인내심은 10초다.
- 플레이어는 주문과 일치하는 완성 요리를 만들어 서빙해야 한다.

현재 구현은 허브 코어 우선 범위다.  
총 방문 인원 제한, 2인 1조 시각 연출 강화, 좌석 확장 규칙은 후속 기획으로 남긴다.

## 7. 프로토타입 테스트 레시피

현재 허브 코어는 아래 3종을 기준으로 테스트한다.
현재 허브 코어 런타임의 메뉴 후보도 이 3종 화이트리스트로 고정한다.

| 레시피명 | 조리기구 | 재료 1 | 재료 2 |
| --- | --- | --- | --- |
| 김치볶음밥 | 후라이팬 | 김치 | 밥 |
| 김치찌개 | 냄비 | 김치 | 고춧가루 |
| 김치전 | 후라이팬 | 김치 | 밀가루 |

현재 프로토타입에서는 기구 조리 완료 결과물을 곧바로 서빙 가능한 완성 요리로 취급한다.

## 8. 성장 보조 공간과 탐험

### scene:room / 종구의 방

- 도구 작업대, 아이템 컬렉션, 책상 위 다이어리가 있는 보조 성장 공간이다.
- 작업대는 도구 수리와 업그레이드의 자리다.
- 컬렉션, 다이어리의 상세 기능은 후속이다.

### scene:beach / 바닷가

- 허브 바깥 첫 탐험 허브다.
- 기본 도구 상자, 갯바위 채집, 부두와 어선, 후속 탐험 방향 암시를 담당한다.

### scene:sea / 바다

- `Beach`에서 이동하는 독립 어망 채집 루프 목표다.

## 9. 후속 탐험 씬 기획

아래 씬은 현재 핵심 허브 코어 밖의 후속 탐험 확장 범위다.

- `DeepForest` / 깊은 숲: 야채 자원, 길 읽기 압박
- `WindHill` / 바람 언덕: 육류 자원, 돌풍 리듬
- `Shortcut` / 지름길: 왕복 피로를 줄이는 개방형 경로
- `AbandonedMine` / 폐광산: 희귀 재료, 고위험/고보상

## 10. 레벨 디자인 관점의 핵심

- `Hub (Close)`는 준비 공간, `Hub (Open)`은 압박 공간으로 읽혀야 한다.
- `Beach`는 첫 외부 맵이므로 방향 읽기가 가장 중요하다.
- 메뉴 선정은 탐험 가치와 운영 난도를 동시에 바꾸는 장치다.
- 인내심 10초는 확정 수치가 아니라 화이트박스 테스트 기준값이다.
