# 종구의 식당 게임 기획 개요

이 문서는 `Jonggu Restaurant` 프로토타입의 게임 기획서 진입점이다.
세부 시스템 기획은 같은 폴더의 다른 문서에서 다룬다. 코드 경로나 빌더 흐름 같은 구현 기준은 `Docs/project/GAME_DOCS_INDEX.md`를 따라 관련 `Docs/project/*`, `Docs/ui/*`, `Docs/scene/*`, `Docs/build/*` 문서를 참고한다.

## 1. 한 줄 소개

2D 탑다운 탐험으로 재료를 모으고, 허브로 돌아와 작은 식당을 운영하는 1인 프로토타입.

## 2. 핵심 컨셉

- 플레이어는 원하는 지역으로 떠나 재료를 모으고, 허브로 돌아와 메뉴를 정하고 영업을 진행한다.
- 영업 결과는 골드와 평판으로 변환되며, 그 결과가 갈 수 있는 지역과 성장 폭을 정한다.
- 체감 비중은 `탐험 8 : 식당 2`를 목표로 한다. 식당은 짧고 가볍게, 탐험은 길고 결정 위주로 유지한다.

## 3. 디자인 기둥

- **작은 결정의 반복** : 인벤토리 한도와 지역 선택이 반복 플레이의 판단을 만든다.
- **단계적 탐험 확장** : 도구 해금과 평판 해금이 새로운 지역을 연다.
- **빠른 영업 사이클** : 메뉴 한 번 결정 → 짧은 영업 → 즉시 결과 확인.
- **읽기 쉬운 화면** : 최소한의 UI로 현재 상태와 다음 행동이 한눈에 들어온다.

## 4. 플레이어 경험 흐름

1. **입문** : `Beach`에서 기본 이동, 채집, 복귀, 영업의 한 사이클을 익힌다.
2. **중반** : `DeepForest`로 진입해 재료 종류를 늘리고 인벤토리 12칸 확장 동기를 얻는다.
3. **후반** : 골드와 재료를 모아 `Lantern`을 해금하고 `AbandonedMine`에 진입한다.
4. **최종** : `WindHill`에서 평판 6 `WindHillShortcut`을 열어 반복 파밍 효율을 높인다.

## 5. 프로토타입 범위

### 포함

- `WASD` 또는 방향키 이동, `E` 상호작용
- 1개 허브 + 4개 탐험 지역
- 재료 전용 인벤토리 슬롯 (`8 → 12 → 16` 확장)
- 도구는 인벤토리 슬롯을 차지하지 않는 영구 해금 아이템
- 단순한 영업 (`serviceCapacity = 3`)
- 데이터 기반 업그레이드 (`골드 + 특정 재료`)
- 허브 전용 창고 보관 시스템

### 제외

- 클릭 이동, 장비 교체 UI, 재료 버리기 시스템
- 복잡한 실시간 식당 운영
- 멀티플레이, 세이브 슬롯, 다국어 UI

## 6. 플레이 가능 씬

| 씬 | 역할 | 핵심 자원 |
| --- | --- | --- |
| `Hub` | 메뉴/창고/업그레이드/포탈 | — |
| `Beach` | 입문 지역 | `Fish`, `Shell`, `Seaweed` |
| `DeepForest` | 중반 지역 | `Mushroom`, `Herb` |
| `AbandonedMine` | 후반 지역 (랜턴 필요) | `Glow Moss` |
| `WindHill` | 최종 지역 (평판 6 지름길) | `Wind Herb` |
