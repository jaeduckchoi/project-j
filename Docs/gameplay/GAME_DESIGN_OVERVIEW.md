# 종구의 식당 게임 기획 개요

이 문서는 `Jonggu Restaurant` 프로토타입의 게임 기획서 진입점이다.
세부 시스템 기획은 같은 폴더의 다른 문서에서 다룬다. 코드 경로나 빌더 흐름 같은 구현 기준은 `Docs/project/GAME_DOCS_INDEX.md`를 따라 관련 `Docs/project/*`, `Docs/ui/*`, `Docs/scene/*`, `Docs/build/*` 문서를 참고한다.

## 1. 한 줄 소개

2D 탑다운 탐험으로 재료를 모으고, 식당 허브에서 오늘의 메뉴를 정한 뒤 `FrontCounter -> BackCounter -> FrontCounter` 조리 루프로 손님 주문을 처리하는 1인 식당 운영 프로토타입.

## 2. 핵심 컨셉

- 플레이어는 작은 섬에 식당을 연 황종구가 되어, 탐험으로 재료를 확보하고 영업으로 골드와 평판을 얻는다.
- 식당 허브는 `Close`와 `Open` 두 상태를 가지며, 상태에 따라 사용할 수 있는 오브젝트와 이동 범위가 달라진다.
- 주방 조리는 `BackCounter`와 `FrontCounter`의 역할 분리로 진행된다.
- `FrontCounter`는 재료와 조리 결과물을 자유롭게 올려두는 공용 작업대이며, 자동 조합 없이 플레이어가 직접 완성 단계를 확정한다.
- `BackCounter`는 도마, 냄비, 후라이팬, 튀김기, 냉장고로 구성되며, 실제 조리 단계와 재료 소모가 발생하는 작업 구역이다.
- 잘못된 조합은 실패 요리를 만들지 않는다. 조건이 맞지 않으면 아무 일도 일어나지 않고 재료는 작업대에 그대로 남는다.

## 3. 디자인 기둥

- **탐험이 만드는 공급 루프** : 바닷가와 외부 지역에서 모은 재료가 냉장고 확인 목록과 조리 입력으로 이어진다.
- **자유 조합형 작업대** : `FrontCounter`는 슬롯 순서보다 구성 조합을 중시하는 작업대이며, 플레이어가 직접 재료를 올리고 회수한다.
- **기구별 다른 조리 리듬** : 도마는 수동 게이지, 냄비/후라이팬/튀김기는 자동 게이지로 구분해 조작 감각을 나눈다.
- **짧고 명확한 영업 사이클** : 메뉴 선정, 주문 확인, 조리, 완성, 서빙, 결과 확인이 한 흐름으로 이어진다.

## 4. 플레이어 경험 흐름

1. **인트로** : 대기업 번아웃을 겪은 황종구가 작은 섬에서 자신만의 식당을 열고 첫 영업을 준비한다.
2. **준비** : `Hub (Close)`에서 냉장고로 보유 재료를 확인하고, 레시피와 메뉴판으로 오늘의 메뉴를 정한다.
3. **탐험** : 외부 출구로 `Beach`(바닷가)에 나가 채집과 이동 루프를 익히고 필요한 재료를 모은다.
4. **영업 개시** : 허브로 돌아와 오늘의 메뉴 3종을 모두 확정한 뒤 `OPEN` UI로 영업을 시작한다.
5. **주문 대응** : 손님 주문을 확인하고 `FrontCounter`에 재료를 올린 뒤, 필요한 `BackCounter` 기구에서 조리한다.
6. **완성 및 서빙** : 조리 결과물을 다시 `FrontCounter`에 모아 완성 요리로 확정하고 손님에게 서빙한다.
7. **성장** : 영업 결과로 얻은 골드와 평판을 바탕으로 작업대, 이동 편의, 탐험 동선을 확장한다.

## 5. 프로토타입 범위

### 포함

- `WASD` 또는 방향키 이동, `E` 상호작용
- 1개 식당 허브 + 6개 탐험맵 씬
- 재료 전용 인벤토리 슬롯 (`8 -> 12 -> 16` 확장)
- 도구는 인벤토리 슬롯을 차지하지 않는 영구 해금 아이템
- 오늘의 메뉴 3종 선정과 `OPEN/CLOSE` 상태 전환
- `FrontCounter` 자유 조합과 `BackCounter` 기구 조리
- 데이터 기반 업그레이드 (`골드 + 특정 재료`)

### 제외

- 클릭 이동, 장비 교체 UI, 재료 버리기 시스템
- 포스기 상세 탭 기획과 리뷰 시스템
- 멀티플레이, 세이브 슬롯, 다국어 UI

## 7. 플레이 가능 씬

| 씬 | 표시 이름 | 크기 | 역할 |
| --- | --- | --- | --- |
| `Hub` | 식당 허브 | `1920 x 1080` | 메뉴 선정, 조리, 영업, 성장, 외부 출구 |
| `Beach` | 바닷가 | `3840 x 2160` | 허브 외부 출구 직후 첫 화면, 중심 연결 필드 |

| `DeepForest` | 깊은 숲 | `2880 x 4320` | 바닷가 우측 연결, 아래쪽 입구에서 위쪽으로 깊어지는 세로형 지역 |
| `WindHill` | 바람 언덕 | `5760 x 2160` | 깊은 숲과 연결되는 가로형 지역, 폐광산으로 이어지는 간접 연결 지점 |
| `Shortcut` | 지름길 | `1920 x 1080` | 바닷가와 바람 언덕을 잇는 골드 소비 개방 지역 |
| `AbandonedMine` | 폐광산 | `3840 x 3240` | 바람 언덕 상단의 독립 고난도/고보상 지역 |
