# 게임플레이 코어 루프

## 1. 최상위 흐름

1. `Hub (Close)`에서 보유 재료와 레시피를 확인한다.
2. `Beach`와 외부 탐험 지역에서 필요한 재료를 모은다.
3. 허브로 돌아와 오늘의 메뉴 3종을 선정한다.
4. `OPEN` UI로 영업을 시작한다.
5. 손님 주문을 확인한다.
6. `FrontCounter`에 재료를 자유롭게 올려 조리 묶음을 준비한다.
7. 준비된 묶음을 `BackCounter` 기구로 가져가 조리한다.
8. 조리 결과물을 다시 `FrontCounter`에 모아 최종 완성한다.
9. 완성 요리를 서빙하고 영업 결과를 확인한다.
10. 영업 종료 후 다시 `Close` 상태로 돌아와 정리, 탐험, 성장을 반복한다.

## 2. 허브 상태 규칙

| 구분 | `Hub (Close)` | `Hub (Open)` |
| --- | --- | --- |
| 냉장고 | 활성 | 활성 |
| `BackCounter` 조리기구 | 비활성 | 활성 |
| `FrontCounter` | 비활성 | 활성 |
| 레시피 | 활성 | 활성 |
| 메뉴판 | 활성 | 비활성 |
| `OPEN` UI | 시작 가능 | 영업 중 상태 표시 |
| `scene:room` 이동 | 가능 | 비활성 |
| `scene:beach` 이동 | 가능 | 비활성 |

- `OPEN` UI는 오늘의 메뉴 3종이 모두 선정되어 있을 때만 영업 시작을 허용한다.
- 메뉴가 모두 정해지지 않았다면 "오늘의 메뉴를 확인하세요." 안내 팝업만 노출한다.
- 영업 중에는 플레이어가 식당 운영에 집중하도록 허브 외부 이동을 막는다.

## 3. 손님 주문 루프

- 손님 AI는 2명씩 1조로 입장해 빈 2인용 테이블에 자동으로 앉는다.
- 가게의 시작 테이블 수는 3개이며, 초기 동시 수용 인원은 최대 6명이다.
- 착석 후 2초가 지나면 오늘의 메뉴 중 하나를 랜덤으로 주문한다.
- 주문 직후 손님 머리 위에 메뉴 아이콘 말풍선이 표시된다.
- 말풍선에는 인내심 게이지가 위에서 아래로 차오르며, 기본 대기 시간은 10초다.
- 플레이어는 주문을 받은 뒤 주방 조리 루프를 수행해 완성 요리를 서빙해야 한다.

## 4. 주방 기본 순서

주방 작업의 공통 순서는 `FrontCounter -> BackCounter -> FrontCounter`로 고정한다.

### 4-1. `FrontCounter` 준비 단계

- `FrontCounter`는 한 번에 최대 5칸의 자유 슬롯을 가진 공용 작업대다.
- 플레이어는 냉장고에서 확인한 보유 재료 중 필요한 항목을 `FrontCounter`에 자유롭게 올릴 수 있다.
- 슬롯 위치 자체는 중요하지 않으며, 판정은 올라간 항목의 구성 조합으로 처리한다.
- 항목 상태는 아래 네 가지로 구분한다.
  - 원재료
  - 손질 재료
  - 조리 완료 결과물
  - 완성 요리
- `FrontCounter`는 자동 조합을 수행하지 않는다. 플레이어가 재료를 준비하고 직접 완성 단계를 실행해야 한다.

### 4-2. `BackCounter` 조리 단계

- 플레이어는 `FrontCounter`에 올려둔 항목 중 필요한 묶음을 집어 각 `BackCounter` 기구로 가져간다.
- `BackCounter`는 외부 구글 시트의 레시피 공정표와 일치하는 유효 입력 묶음만 조리 시작을 허용한다.
- 조리를 시작하는 순간 해당 단계의 입력 재료를 즉시 소모한다.
- 각 기구는 동시에 1개의 작업만 가진다.
- 도마는 수동 게이지 조리다.
  - 조리 시작 후 게이지가 모두 찰 때까지 플레이어가 상호작용을 유지해야 한다.
  - 작업 중에는 이동할 수 없다.
- 냄비, 후라이팬, 튀김기는 자동 게이지 조리다.
  - 조리 시작 후 자동으로 게이지가 찬다.
  - 진행 중 플레이어는 이동할 수 있다.
- 조리 완료 시 기구 위에는 1개의 조리 결과물만 남는다.

### 4-3. `FrontCounter` 완성 단계

- 플레이어는 `BackCounter`에서 나온 조리 결과물을 다시 `FrontCounter`로 가져온다.
- `FrontCounter`는 현재 작업대에 올라온 전체 항목을 기준으로 최종 조합 가능 여부를 검증한다.
- 최종 완성은 자동이 아니라 플레이어의 `완성하기` 상호작용으로 확정한다.
- 조합이 정확히 맞으면 완성 요리가 생성된다.
- 조합이 정확히 맞지 않으면 아무 일도 일어나지 않고, 올라간 항목은 그대로 유지된다.
- 실패 요리, 오염된 요리, 잘못된 부산물은 생성하지 않는다.

## 5. 꼬임 방지 규칙

- `FrontCounter`는 자유 조합판이지만 자동 조합은 하지 않는다.
- `BackCounter`는 유효하지 않은 입력 조합을 받으면 조리를 시작하지 않는다.
- 원재료 소모는 `BackCounter` 조리 시작 시점에만 발생한다.
- 최종 완성 판정은 슬롯 순서가 아니라 항목 목록과 상태 태그 기준으로 처리한다.
- 잘못 올린 재료는 유지되므로 플레이어가 다시 집어서 정리할 수 있다.
- 동일한 재료 구성의 중복 레시피는 외부 레시피 공정표에서 관리하지 않는 것을 전제로 한다.
- 단일 단계 요리도 `FrontCounter`의 최종 완성 단계를 거쳐야 서빙 가능한 완성 요리가 된다.

## 6. 테스트 레시피 예시

문서와 플레이 흐름 설명은 아래 3종을 기준으로 통일한다.
세부 공정 단계와 추가 레시피는 외부 구글 시트의 레시피 공정표를 정본으로 사용한다.

| 레시피명 | 조리법 | 재료 1 | 재료 2 |
| --- | --- | --- | --- |
| 김치볶음밥 | 프라이팬 | 김치 | 밥 |
| 김치찌개 | 냄비 | 김치 | 고춧가루 |
| 김치전 | 프라이팬 | 김치 | 밀가루 |

- 김치볶음밥 : `FrontCounter`에 김치와 밥을 올림 -> 후라이팬 조리 시작 -> 결과물을 `FrontCounter`에 올림 -> 완성하기 -> 김치볶음밥 생성
- 김치찌개 : `FrontCounter`에 김치와 고춧가루를 올림 -> 냄비 조리 시작 -> 결과물을 `FrontCounter`에 올림 -> 완성하기 -> 김치찌개 생성
- 김치전 : `FrontCounter`에 김치와 밀가루를 올림 -> 후라이팬 조리 시작 -> 결과물을 `FrontCounter`에 올림 -> 완성하기 -> 김치전 생성

## 7. 탐험 이동 축

- `Beach`는 식당 허브 외부 출구 직후의 첫 화면이자 중심 연결 노드다.
- `Sea`는 바닷가 좌측 어선으로 이동하는 독립 어망 채집 공간이다.
- `DeepForest`는 바닷가 우측에서 진입하며, 아래쪽 입구에서 위쪽으로 깊어지는 세로형 지역이다.
- `WindHill`은 깊은 숲과 연결된 가로형 지역이며, 상단에서 `AbandonedMine`으로 이어진다.
- `Shortcut`은 초반에는 막혀 있고, 골드를 소비해 `Beach`와 `WindHill` 사이를 여는 편의 경로다.

## 8. API 연동 기준

- 원격 세션과 스냅샷 반영은 `JongguApiSession`, `GameManager`를 기준으로 유지한다.
- 레시피 메타데이터와 기본 재료 구성은 서버 bootstrap `recipes`, `ingredients`를 기준으로 유지한다.
- 자유 조합형 `FrontCounter`의 상세 공정 단계와 시그니처는 외부 레시피 공정표 정본을 따른다.
- 안내 문구, 포탈 잠금, 팝업 일시정지 회귀는 현재 런타임 코드 기준으로 직접 확인한다.
