# 식당 영업과 성장 기획

## 1. 허브의 역할

허브는 준비, 영업, 조리, 탐험 출발, 성장의 중심 공간이다.

- `Close`: 준비와 탐험 출발
- `Open`: 주문 대응과 조리 압박

현재 허브 코어의 주방 역할은 아래처럼 나눈다.

- `냉장고`: 보유 재료 확인
- `CookingUtensils`: 재료 선택과 조리 시작
- `PassCounter`: 결과물 적재와 회수

## 2. 허브 상태와 상호작용

### `Hub (Close)`

- 냉장고: 활성
- 레시피: 활성
- 메뉴판: 활성
- 포스기: 후속 기획 위치 유지
- PassCounter: 비활성
- CookingUtensils: 비활성
- `scene:room`: 활성
- `scene:beach`: 활성

### `Hub (Open)`

- 냉장고: 활성
- 레시피: 활성
- 메뉴판: 비활성
- 포스기: 비활성
- PassCounter: 활성
- CookingUtensils: 활성
- `scene:room`: 차단
- `scene:beach`: 차단

## 3. 허브 오브젝트 상세

### 3-1. 냉장고

- 보유 재료 확인용 팝업이다.
- 기본 재료는 무한, 탐험 재료는 현재 보유 수량 기준이다.
- 현재 단계에서는 보관/인출보다 확인에 집중한다.

### 3-2. 레시피

- 해금된 요리와 필요 재료, 조리기구, 판매 가격을 확인한다.

### 3-3. 메뉴판

- 오늘의 메뉴 3칸을 직접 채운다.
- 3칸이 모두 채워져야 영업을 시작할 수 있다.
- 현재 허브 코어 런타임의 메뉴 후보는 `김치볶음밥`, `김치찌개`, `김치전` 3종이다.

### 3-4. 포스기

- 현재는 자리를 잡아 두는 수준이다.
- 매출, 리뷰, 업그레이드 상세는 후속이다.

### 3-5. PassCounter

- 조리 완료품을 잠시 올려두는 작업대다.
- 결과물을 다시 집어 테이블로 가져갈 수 있다.
- 현재 슬롯 수는 6칸이다.

### 3-6. CookingUtensils

- 도마, 냄비, 후라이팬, 튀김기로 구성된다.
- 유효한 재료 조합일 때만 조리가 시작된다.
- 조리 시작 시 재료가 소모된다.
- 잘못된 조합은 실패 요리 없이 그대로 멈춘다.

## 4. 손님 주문과 서빙

- 주문은 오늘의 메뉴 중 하나에서 나온다.
- 착석 후 2초 뒤 주문이 뜬다.
- 기본 인내심은 10초다.
- 완성 요리를 손에 들고 해당 테이블과 상호작용하면 서빙된다.

현재 구현은 허브 코어 우선 범위다.  
총 12명 방문 제한, 2인 1조 연출, 좌석 확장 규칙은 후속으로 둔다.

## 5. 종구의 방

`scene:room`은 보조 성장 공간이다.

- 도구 작업대: 수리와 업그레이드
- 아이템 컬렉션: 후속
- 다이어리: 후속

영업 중 접근 차단은 현재 허브 코어 활성 규칙이다.

## 6. 인벤토리와 성장

- 탐험으로 재료를 모으고 허브에서 소모한다.
- 골드는 영업 보상과 성장 비용에 쓰인다.
- 이동 편의, 탐험 효율, 도구 성장, 허브 편의 확장은 장기 성장 축이다.
