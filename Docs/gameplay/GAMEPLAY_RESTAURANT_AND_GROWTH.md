# 식당 영업과 성장 기획

## 1. 허브의 역할

허브는 영업, 준비, 성장, 탐험 출발이 모두 이뤄지는 메인 공간이다.
플레이어는 허브를 직접 걸어 다니며 재료를 확인하고, 오늘의 메뉴를 정하고, 영업 상태를 전환하고, 조리를 진행하고, 외부 출구로 `Beach`(바닷가)에 나간다.

허브의 주방 구역은 아래 두 역할로 나뉜다.

- `BackCounter` : 냉장고, 도마, 냄비, 후라이팬, 튀김기
- `FrontCounter` : 재료 자유 배치, 조리 결과물 집결, 최종 완성 작업대

## 2. 허브 상태와 상호작용 지점

### `Hub (Close)`

- 냉장고 : 활성
- 레시피 : 활성
- 메뉴판 : 활성
- `BackCounter` 조리기구 : 비활성
- `FrontCounter` : 비활성
- `scene:room` 이동 : 활성
- `scene:beach` 이동 : 활성

### `Hub (Open)`

- 냉장고 : 활성
- 레시피 : 활성
- 메뉴판 : 비활성
- `BackCounter` 조리기구 : 활성
- `FrontCounter` : 활성
- `scene:room` 이동 : 비활성
- `scene:beach` 이동 : 비활성

## 3. 허브 상호작용 상세

### 3-1. 냉장고

- `E` 상호작용으로 팝업을 연다.
- 현재 보유 중인 요리 재료를 자유 목록 형태로 확인한다.
- 초기 버전은 탭 분류 없이 전체 보유 재료를 한 화면에서 보여 준다.
- 기본 재료 `김치`, `밥`, `고춧가루`, `밀가루`는 횟수 제한 없이 상시 사용 가능하다.
- 탐험으로 획득한 해산물, 채소, 기타 재료도 같은 목록에 함께 표시한다.
- 냉장고는 보관/인출 기능보다 "현재 조리에 사용할 수 있는 재료 확인"에 초점을 둔다.

### 3-2. 레시피

- `E` 상호작용으로 팝업을 연다.
- 해금한 요리 아이콘 목록을 정렬해 보여 준다.
- 요리 선택 시 아래 정보를 확인할 수 있다.
  - 레시피 이름
  - 필요한 재료
  - 사용해야 하는 주요 조리기구
  - 간단한 요리 소개
  - 판매 가격
- 상세 공정 단계와 조합 시그니처는 외부 구글 시트의 레시피 공정표를 정본으로 참조한다.

### 3-3. 메뉴판

- `E` 상호작용으로 팝업을 연다.
- 오늘의 메뉴 3칸을 직접 지정한다.
- 빈칸을 선택하면 해금한 음식 중 원하는 메뉴를 배치할 수 있다.
- 3칸을 모두 확정하고 닫으면 메뉴판에 오늘의 메뉴가 반영된다.
- 오늘의 메뉴 3종이 모두 확정되어야 `OPEN` UI가 영업 시작을 허용한다.

### 3-4. `OPEN` UI

- 화면 오른쪽의 `OPEN` UI를 누르면 영업 시작 확인 팝업이 뜬다.
- 오늘의 메뉴 3종이 모두 정해져 있다면 "오늘의 메뉴를 확인하세요. 영업을 시작하시겠습니까?" 팝업 후 영업이 시작된다.
- 메뉴가 모두 정해지지 않았다면 "오늘의 메뉴를 확인하세요." 팝업만 표시된다.

### 3-5. `scene:room`

- 종구의 방에는 도구 작업대, 아이템 컬렉션, 책상 위 다이어리가 배치된다.
- 작업대는 내구도가 깎인 도구 수리와 다음 단계 업그레이드에 사용한다.
- 아이템 컬렉션과 다이어리는 후속 기능으로 남긴다.

## 4. `FrontCounter`와 `BackCounter` 조리 규칙

### 4-1. `FrontCounter`

- `FrontCounter`는 최대 5칸의 자유 슬롯을 가진 공용 작업대다.
- 플레이어는 조리에 쓸 재료와 조리 결과물을 자유롭게 올리고 뺄 수 있다.
- 판정 기준은 슬롯 위치가 아니라 현재 올라간 항목의 구성 조합이다.
- 작업대 위 항목은 다음 상태 중 하나를 가진다.
  - 원재료
  - 손질 재료
  - 조리 완료 결과물
  - 완성 요리
- `FrontCounter`는 자동 조합하지 않는다.
- 플레이어가 `완성하기` 상호작용을 수행했을 때만 최종 조합을 검증한다.
- 조합이 정확히 맞으면 완성 요리가 생성된다.
- 조합이 맞지 않으면 아무 일도 일어나지 않고, 현재 올려둔 항목은 그대로 유지된다.

### 4-2. `BackCounter`

- `BackCounter`는 냉장고, 도마, 냄비, 후라이팬, 튀김기 구역으로 구성된다.
- 플레이어는 `FrontCounter`에서 필요한 재료 묶음을 집어 각 기구로 가져간다.
- 각 기구는 외부 구글 시트의 레시피 공정표에서 유효한 입력 묶음일 때만 조리를 시작한다.
- 조리 시작 시 해당 단계의 입력 재료를 즉시 소모한다.
- 각 기구는 동시에 1개의 작업만 처리한다.
- 조리 완료 시 기구 위에는 1개의 결과물만 남고, 플레이어가 이를 다시 `FrontCounter`로 옮긴다.

### 4-3. 기구별 게이지 규칙

- 조리 시작 시 해당 기구 위에 게이지가 표시된다.
- 도마
  - 손질용 기구다.
  - 플레이어가 게이지가 끝날 때까지 직접 상호작용을 유지해야 한다.
  - 작업 중에는 이동할 수 없다.
- 냄비
  - 자동 게이지 조리다.
  - 작업 중 이동할 수 있다.
- 후라이팬
  - 자동 게이지 조리다.
  - 작업 중 이동할 수 있다.
- 튀김기
  - 자동 게이지 조리다.
  - 작업 중 이동할 수 있다.
- 게이지 UI의 기본 시각 방향은 첨부 예시 이미지를 기준으로 조리 상태를 직관적으로 전달하는 형태를 유지한다.

## 5. 꼬임 방지 규칙

- `FrontCounter`는 자유 조합판이지만 자동 완성하지 않는다.
- `BackCounter`는 유효하지 않은 입력 묶음을 받으면 조리를 시작하지 않는다.
- 최종 조합 판정은 슬롯 순서가 아니라 항목 목록과 상태 태그 기준으로 처리한다.
- 잘못 올린 재료는 그대로 유지되므로 플레이어가 다시 집어서 정리할 수 있다.
- 실패 요리나 오염된 요리는 생성하지 않는다.
- 동일한 시그니처를 가진 중복 레시피는 외부 레시피 공정표에서 허용하지 않는 것을 전제로 한다.
- 단일 단계 요리도 `FrontCounter`의 최종 완성 단계를 거쳐야 서빙 가능한 완성 요리가 된다.

## 6. 손님 주문과 서빙

- 손님 AI는 2명씩 1조로 입장해 빈 테이블에 자동으로 앉는다.
- 기본 테이블 수는 3개이며, 시작 시 최대 동시 수용 인원은 6명이다.
- 착석 후 2초가 지나면 오늘의 메뉴 중 하나를 랜덤으로 주문한다.
- 손님 머리 위 말풍선에 주문한 메뉴 아이콘이 표시된다.
- 주문 직후 인내심 게이지가 위에서 아래로 차오르며, 기본 대기 시간은 10초다.
- 플레이어는 주문과 일치하는 완성 요리를 서빙해야 한다.
- 서빙 성공 시 영업 결과와 성장 재화가 반영된다.

## 7. 프로토타입 테스트 레시피

문서와 플레이 흐름 설명은 아래 3종을 기준으로 통일한다.

| 레시피명 | 조리법 | 재료 1 | 재료 2 |
| --- | --- | --- | --- |
| 김치볶음밥 | 프라이팬 | 김치 | 밥 |
| 김치찌개 | 냄비 | 김치 | 고춧가루 |
| 김치전 | 프라이팬 | 김치 | 밀가루 |

### 예시 흐름

- 김치볶음밥 : `FrontCounter`에 김치와 밥 배치 -> 후라이팬 조리 시작 -> 결과물을 `FrontCounter`에 올림 -> `완성하기` -> 김치볶음밥 생성
- 김치찌개 : `FrontCounter`에 김치와 고춧가루 배치 -> 냄비 조리 시작 -> 결과물을 `FrontCounter`에 올림 -> `완성하기` -> 김치찌개 생성
- 김치전 : `FrontCounter`에 김치와 밀가루 배치 -> 후라이팬 조리 시작 -> 결과물을 `FrontCounter`에 올림 -> `완성하기` -> 김치전 생성

## 8. 인벤토리, 경제, 성장

### 인벤토리

| 단계 | 슬롯 수 |
| --- | --- |
| 시작 | 8 |
| 1차 확장 | 12 |
| 2차 확장 | 16 |

- 재료 전용 슬롯 구조이며 도구는 슬롯을 차지하지 않는다.
- 슬롯 압박은 탐험 재료 선택과 영업 준비 판단을 만드는 의도된 요소다.

### 골드와 평판

- 골드는 영업 결과와 업그레이드 비용에 사용한다.
- 평판은 성장 폭과 편의 해금에 연결된다.
- `Shortcut`(지름길)은 초반에는 막혀 있으며 골드를 소비해 개방한다.

### 업그레이드 비용

| 항목 | 비용 |
| --- | --- |
| 인벤토리 12칸 확장 | `Gold 30 + Shell x3` |
| 인벤토리 16칸 확장 | `Gold 65 + Herb x4` |
| 랜턴 해금 | `Gold 45 + Mushroom x2` |

## 9. 허브 팝업과 시간 정지

- 팝업 종류 : `Recipe`, `Materials`, `Cooking`, `Upgrade`
- 팝업이 열리면 게임 시간이 멈춘다.
- `Esc` 또는 `PopupCloseButton`으로 닫으면 이전 시간 흐름이 복구된다.
- 일시 정지 동작은 `PopupPauseStateUtility`가 관리한다.
