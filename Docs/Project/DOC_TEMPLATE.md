# 문서 템플릿

새 정본 문서를 만들거나 기존 문서를 재편할 때는 아래 골격을 먼저 채운다.

## 역할

이 문서가 무엇의 정본인지 한두 문장으로 적는다.

## 이 문서를 읽는 시점

- 어떤 작업에서 가장 먼저 열어야 하는지 적는다.
- 함께 읽을 상위, 하위 문서가 있으면 Markdown 링크로 연결한다.

## 정본 범위

- 이 문서가 단독 소유하는 규칙, 수치, 표, 경로를 적는다.
- 다른 문서가 정본인 내용은 복제하지 않고 링크만 남긴다.

## 함께 수정할 항목

- 코드, 씬, 자산, 다른 문서 중에서 결합되어 같이 바뀌어야 하는 항목을 적는다.

## 검증

- 정적 검색, Unity 확인, 수동 점검 등 이 문서 기준으로 확인해야 하는 항목을 적는다.
