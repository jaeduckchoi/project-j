# 바닷가 화이트박스 기준

이 문서는 `Beach.unity`의 맵 크기, 화면 연출 목표, 현재 프로젝트 정합성 점검 결과를 정리한 정본 문서다.  
바닷가의 최종 배치 정본은 씬 직렬화 값이고, 현재 탐험 허브의 화면 기준은 이 문서를 따른다.

## 1. 화면 기준 정본

- 화면 연출 목표: `1920x1080`
- 목표 비율: `16:9`
- 맵 크기: `3840x2160`
- 화면 수: `2x2 화면`
- 월드 기준 해석: `64x36`

바닷가는 전체를 한 번에 다 보여주는 맵이 아니라, 이동하면서 방향을 읽는 탐험 허브로 연출한다.  
한 화면 안에서는 현재 위치의 작업 지점만 읽히고, 이동하면서 부두, 갯바위, 숲 방향을 차례로 파악하는 구성을 목표로 둔다.

## 2. 현재 프로젝트 정합성 점검

### 판정

`정합`

### 근거

- `Beach.unity`의 카메라는 `orthographic size 9`를 사용한다.
- `Beach.unity`의 `CameraBounds`는 `64x36`이다.
- `Beach.unity`의 `BeachMovementBounds`도 `64x36`이다.
- `Beach.unity`의 UI 기준 해상도는 `1920x1080`이다.

16:9 화면에서 `orthographic size 9`는 한 화면 가시 영역을 약 `32x18`로 만든다.  
현재 `Beach`의 `64x36` authored bounds는 그 기준에서 정확히 `2x2 화면` 크기와 대응된다.

## 3. 공간 읽기 목표

- 집 앞 도구 상자는 초반 입문 동선을 바로 알려야 한다.
- 아래쪽 갯바위는 첫 채집 루프로 읽혀야 한다.
- 좌측 부두와 어선은 바다 진출 방향을 분명하게 보여줘야 한다.
- 우측 숲 방향은 후속 탐험 확장 루트로 암시되어야 한다.

`Beach`는 첫 외부 맵이므로 복잡한 퍼즐보다 방향 읽기와 확장 기대감을 우선한다.

## 4. 타일 원본 기준 해석 메모

- 이 문서의 `3840x2160`은 `화면 기준 px`다.
- 현재 프로젝트의 타일 import 계약은 `64px` 타일 기준을 유지한다.
- 타일 원본 기준 px와 화면 기준 px를 같은 의미로 취급하지 않는다.
- 타일 원본 기준으로 exact matching하려면 `60px` 타일 계약 전환이 필요하다. 이번 범위에는 포함하지 않는다.
