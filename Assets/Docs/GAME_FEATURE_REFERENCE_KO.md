# 종구의 식당 기능 통합 문서

## 1. 게임 개요

`종구의 식당`은 2D 탑다운 탐험-운영 게임으로 정리된 현재 구현 문서다.
핵심 감각은 `탐험에서 재료를 모으고`, `허브로 복귀해`, `간단한 장사를 돌리고`, `결과를 성장으로 환원하는 하루 루프`에 있다.

현재 구현 기준 플레이 흐름은 아래 순서로 이어진다.

1. 허브에서 현재 상태 확인
2. 오전 탐험 지역 진입
3. 자원 채집과 인벤토리 관리
4. 허브 복귀
5. 메뉴 선택
6. 장사 실행
7. 골드 / 평판 정산
8. 업그레이드 또는 다음 날 전환

## 2. 게임 특징

### 탐험 중심 구조

- 전체 체감 비중은 `탐험 8 : 장사 2` 에 가깝게 맞춰져 있다.
- 장사는 실시간 운영보다 결과 정산 중심으로 단순화했다.
- 플레이어는 탐험 지역마다 다른 위험과 자원 밀도를 체감하도록 설계되어 있다.

### 허브 순환 구조

- 허브는 단순 메뉴 화면이 아니라 직접 걸어 다니는 공간이다.
- 창고, 작업대, 메뉴판, 영업대, 지역 포탈이 각각 월드 상호작용 오브젝트로 배치되어 있다.
- 복귀 후 `정리 -> 선택 -> 영업 -> 정산` 흐름이 한 장소 안에서 이어진다.

### 성장 구조

- 골드만이 아니라 특정 자원도 업그레이드 비용으로 사용한다.
- 인벤토리 확장과 랜턴 해금이 중반 이후 탐험 확장 조건으로 이어진다.
- 평판은 바람 언덕 숏컷 같은 편의성 해금에 사용된다.

## 3. 지역별 특징

### Hub

- 허브 이름은 `종구의 식당`이다.
- 메뉴 선택대, 영업대, 창고, 작업대, 바닷가 / 깊은 숲 / 폐광산 / 바람 언덕 포탈이 들어간다.
- 선택형 창고 패드와 폐광산 포탈이 직렬화되어 있고, 런타임 안전망도 남겨뒀다.

### Beach

- 입문 탐험 지역이다.
- `생선`, `조개`, `해초` 수집으로 기본 상호작용을 익힌다.
- 위험 지형은 빼고 기본 이동과 채집 루프 확인에 집중한다.

### DeepForest

- 중반 탐험 지역이다.
- `버섯`, `약초` 수집과 감속 지형이 들어간다.
- 인벤토리 확장과 랜턴 해금 재료 흐름을 이어주는 역할을 한다.

### AbandonedMine

- 후반 탐험 지역이다.
- `GlowMoss` 채집과 랜턴 요구 조건이 연결되어 있다.
- `MineDarkness` 와 `MineLooseRubble` 로 시야 / 이동 리스크를 준다.
- 현재는 플레이 가능한 초기 구현 버전이며, 미술 / 배치는 추가 다듬기 여지가 있다.

### WindHill

- 최종 탐험 지역이다.
- 강풍 존이 주기적으로 플레이어를 밀어낸다.
- 평판 6 이상이면 `WindHillShortcut` 을 사용할 수 있다.

## 4. 시스템별 기능

### 플레이어와 상호작용

- `PlayerController`
  이동 입력과 상호작용 입력을 처리한다.
- `InteractionDetector`
  가까운 상호작용 대상을 감지하고 현재 프롬프트를 고른다.
- `IInteractable`
  월드 오브젝트 상호작용 규약을 통일한다.

### 자원과 채집

- `ResourceData`
  자원 이름, 설명, 아이콘, 희귀도, 판매가를 보관한다.
- `GatherableResource`
  필요한 도구를 검사하고 채집 시 인벤토리에 자원을 넣는다.
- 차단 상태에서도 입력을 받아 이유를 안내하도록 맞춰져 있다.

### 도구와 진입 조건

- 도구는 인벤토리를 차지하지 않는 상시 해금 구조다.
- 현재 사용 도구는 `갈퀴`, `낚시대`, `낫`, `랜턴` 이다.
- 랜턴은 폐광산 진입과 어둠 구간 통과 조건으로 연결되어 있다.

### 인벤토리와 창고

- 인벤토리는 자원 전용 슬롯 구조다.
- 시작 8칸, 확장 후 12칸, 16칸까지 늘어난다.
- 창고는 맡기기 / 꺼내기 / 선택 품목 입출고를 지원한다.
- 허브에서만 접근 가능한 구조로 유지했다.

### 장사와 메뉴

- `RecipeData`
  메뉴 이름, 설명, 판매가, 평판 변화, 필요 재료를 정의한다.
- `RestaurantManager`
  현재 선택 메뉴, 조리 가능 여부, 재료 소모, 결과 정산을 담당한다.
- 선택 메뉴 설명과 필요 재료, 현재 보유량, 결과 텍스트가 UI에 반영된다.

### 경제와 업그레이드

- `EconomyManager`
  골드와 평판을 관리한다.
- `UpgradeManager`
  인벤토리 확장과 도구 해금을 처리한다.
- 현재 가능한 업그레이드를 우선 선택하도록 로직을 정리했다.

### 하루 흐름

- `DayCycleManager`
  오전 탐험, 오후 장사, 정산, 다음 날 전환을 관리한다.
- 탐험 스킵, 장사 스킵, 다음 날 버튼이 단계별로 열리고 닫힌다.
- 지역 입장 힌트와 1회성 안내 문구를 함께 관리한다.

### UI와 텍스트 표시

- 인벤토리, 창고, 업그레이드, 메뉴, 결과, 단계, 프롬프트 UI가 들어가 있다.
- 최근 수정으로 에디터 플레이 기준 텍스트 패널 대비와 글자 크기, 여백, 줄간격을 다시 조정했다.
- TextMesh Pro 기본 폰트는 `MalgunGothic SDF` 기준으로 맞춰 한글 표시가 안정되도록 정리했다.

### 데이터 복구와 런타임 안전망

- `GeneratedGameDataLocator`
  생성 에셋 참조가 비어 있을 때 기본 자원 / 레시피를 다시 찾는다.
- `GeneratedGameDataManifest`
  빌드에서도 generated 에셋 참조가 유지되도록 묶어 둔다.
- `PrototypeSceneRuntimeAugmenter`
  씬이 덜 생성된 경우 허브 패드, 포탈, 숏컷, 위험 구간을 런타임에서 메운다.

## 5. 텍스트와 한글 처리 기준

- 모든 주요 UI 텍스트는 TextMesh Pro 기반이다.
- 기본 폰트는 `Assets/Generated/Fonts/MalgunGothic SDF.asset` 기준으로 통일했다.
- TMP 설정에서 한글 줄바꿈 규칙을 사용하도록 맞췄다.
- UI 패널과 텍스트 배치도 런타임에서 다시 조정해 밝은 배경 / 어두운 배경 어느 쪽에서도 읽기 쉽게 보강했다.

## 6. 현재 구현 상태 요약

### 완료된 범위

- 허브와 4개 탐험 지역 연결
- 채집 / 귀환 / 메뉴 선택 / 장사 / 정산 / 다음 날 루프
- 인벤토리 / 창고 / 업그레이드 / 평판 / 숏컷
- 생성 데이터 복구와 런타임 안전망
- 텍스트 주석 정리와 기본 문서화

### 남은 실질 작업

- Unity 플레이 기준 실제 밸런스 조정
- 텍스트와 버튼 배치의 미세 조정
- 탐험 지역 미술 / 충돌 / 동선 다듬기
- 안전망 로직 의존 축소
