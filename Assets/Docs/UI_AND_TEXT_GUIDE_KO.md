# 종구의 식당 UI 및 텍스트 문서

## 1. UI 구성 목적

UI는 탐험을 방해하지 않으면서 현재 상태를 빠르게 읽을 수 있게 정리하는 것을 목표로 한다.
허브와 탐험 모두 TextMesh Pro 기반으로 구성되어 있다.

## 2. 주요 UI 블록

### 좌상단

- 현재 지역 이름
- 골드 / 평판

### 좌하단

- 인벤토리
- 창고

### 우상단

- 선택 메뉴
- 장사 결과
- 업그레이드 요약

### 상단 중앙

- 현재 날짜와 단계
- 안내 문구

### 하단 중앙

- 현재 상호작용 프롬프트

### 우하단 버튼

- 탐험 스킵
- 장사 스킵
- 다음 날

## 3. 텍스트 가독성 보강 내용

- UI 패널 배경 알파와 크기를 다시 조정했다.
- 텍스트 크기, 여백, 줄간격을 런타임에서 다시 정리한다.
- 버튼 라벨도 같은 기준으로 다시 스타일링한다.
- 월드 라벨은 굵게 처리하고 외곽선을 넣어 바닥색에 덜 묻히게 했다.

## 4. 한글 처리 기준

- TMP 기본 폰트는 `MalgunGothic SDF` 기준으로 통일했다.
- TMP 기본 설정에서 한글 줄바꿈 규칙을 사용하도록 켰다.
- generated 폰트가 다시 만들어져도 같은 경로를 기본 폰트로 쓰도록 맞췄다.

## 5. 관련 핵심 파일

- `Assets/Scripts/UI/UIManager.cs`
  런타임 UI 갱신과 텍스트 가독성 보강 처리
- `Assets/Editor/JongguMinimalPrototypeBuilder.cs`
  새 씬 생성 시 기본 UI 패널과 텍스트 배치 생성
- `Assets/TextMesh Pro/Resources/TMP Settings.asset`
  TMP 기본 폰트와 한글 줄바꿈 설정

## 6. 확인할 포인트

- 에디터 Play 시 한글이 깨지지 않는지
- 탐험 지역 밝기와 상관없이 프롬프트가 읽히는지
- 창고 / 메뉴 / 업그레이드 다중 줄 텍스트가 겹치지 않는지
- 월드 라벨이 바닥색에 묻히지 않는지

## 7. 현재 상태

- 런타임 보정과 빌더 기본값 수정이 같이 들어가 있다.
- 기존 씬도 Play 시에는 가독성 보정이 적용된다.
- 빌더를 다시 돌리면 새 기본값도 씬 직렬화에 반영된다.
