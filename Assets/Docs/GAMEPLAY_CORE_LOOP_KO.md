# 종구의 식당 핵심 루프 문서

## 1. 핵심 플레이 구조

이 게임의 중심은 `하루 단위 탐험-운영 루프`다.
플레이어는 직접 이동하며 재료를 모으고, 허브로 돌아와 장사를 처리한 뒤, 수익과 해금으로 다음 탐험을 준비한다.

현재 구현 기준 기본 흐름은 아래와 같다.

1. 허브에서 상태 확인
2. 오전 탐험 지역 진입
3. 채집과 인벤토리 관리
4. 허브 복귀
5. 메뉴 선택
6. 장사 실행
7. 결과 정산
8. 업그레이드 또는 다음 날 전환

## 2. 플레이어가 느껴야 하는 감각

- 오늘 어느 지역을 갈지 정하는 기대감
- 제한된 인벤토리 안에서 무엇을 챙길지 고르는 판단감
- 더 들어갈지 돌아갈지 정하는 탐험 긴장감
- 재료가 메뉴와 골드로 바뀌는 만족감
- 업그레이드 후 더 깊은 지역으로 나가는 성장감

## 3. 단계별 역할

### 허브 준비

- 현재 골드, 평판, 보유 재료 확인
- 창고 정리
- 업그레이드 가능 여부 확인
- 어느 지역으로 갈지 선택

### 오전 탐험

- 지역별 자원 채집
- 도구 요구 조건 확인
- 감속 / 강풍 / 어둠 같은 환경 요소 체감
- 인벤토리 한계 안에서 수집 판단

### 허브 복귀 후 장사

- 메뉴 선택
- 필요 재료 확인
- 판매 결과 계산
- 골드와 평판 획득

### 정산과 다음 날

- 결과 텍스트 확인
- 필요하면 업그레이드 적용
- 다음 날로 전환

## 4. 현재 루프에서 중요한 연결점

- 탐험에서 얻은 자원은 바로 장사 재료로 연결된다.
- 장사 결과는 골드 / 평판 / 업그레이드 가능 상태로 다시 이어진다.
- 랜턴 해금과 평판은 신규 지역 접근성으로 연결된다.
- UI는 현재 단계에 따라 버튼과 안내 문구를 바꿔준다.

## 5. 현재 구현 상태

- 오전 탐험 / 오후 장사 / 정산 / 다음 날 전환은 코드 기준으로 연결되어 있다.
- 탐험 스킵, 장사 스킵 버튼도 단계에 따라 열리도록 맞춰져 있다.
- 루프 자체는 플레이 가능한 수준으로 이어지지만, 실제 밸런스 수치는 플레이 테스트 후 조정이 필요하다.
