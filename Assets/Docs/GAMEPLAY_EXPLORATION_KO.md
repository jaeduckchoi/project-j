# 종구의 식당 탐험 기능 문서

## 1. 탐험 시스템 개요

탐험은 이 게임의 핵심 비중을 차지한다.
플레이어는 직접 이동하면서 지역 자원을 채집하고, 도구 요구 조건과 환경 리스크를 확인하며 복귀 시점을 판단한다.

## 2. 공통 탐험 요소

### 이동과 상호작용

- `WASD` 또는 방향키로 이동
- `E` 키로 상호작용
- 가까운 상호작용 대상은 `InteractionDetector` 가 고른다.

### 채집

- 채집 오브젝트는 `GatherableResource` 로 처리한다.
- 필요한 도구가 없으면 채집이 막히고 이유를 안내한다.
- 채집 성공 시 자원이 인벤토리에 들어간다.

### 도구

- `갈퀴`
- `낚시대`
- `낫`
- `랜턴`

도구는 인벤토리를 차지하지 않고, 해금 후 상시 보유 상태로 관리한다.

## 3. 지역별 탐험 특징

### Beach

- 입문 지역
- 자원: `생선`, `조개`, `해초`
- 목적: 기본 이동과 채집 루프 학습
- 특징: 위험 요소 최소화

### DeepForest

- 중반 지역
- 자원: `버섯`, `약초`
- 목적: 업그레이드 재료 흐름 연결
- 특징: 감속 지형, 경로 선택

### AbandonedMine

- 후반 지역
- 자원: `GlowMoss`
- 목적: 랜턴 해금 후 진입하는 고위험 지역
- 특징: 어둠 구간, 잔해 감속, 랜턴 요구 조건

### WindHill

- 최종 지역
- 자원: `WindHerb`
- 목적: 반복 수급과 평판 보상 확인
- 특징: 주기성 강풍, 평판 기반 숏컷

## 4. 환경 위험 요소

### 감속 구간

- `MovementModifierZone` 으로 처리한다.
- 늪, 잔해처럼 지나가기 불편한 지형을 표현한다.

### 어둠 구간

- `DarknessZone` 으로 처리한다.
- 랜턴이 없으면 이동 배율이 낮아진다.

### 강풍 구간

- `WindGustZone` 으로 처리한다.
- 주기에 따라 플레이어를 한 방향으로 밀어낸다.

## 5. 탐험에서 확인할 포인트

- 자원이 의도한 도구 조건으로 막히는지
- 막힌 이유 안내 문구가 뜨는지
- 귀환 포탈이 허브 복귀 흐름을 끊지 않는지
- 위험 구간이 시각적으로 구분되는지
- 인벤토리 압박이 수집 판단으로 이어지는지

## 6. 현재 구현 상태

- `Beach`, `DeepForest`, `AbandonedMine`, `WindHill` 네 지역이 연결되어 있다.
- 랜턴 조건, 강풍, 감속, 어둠, 숏컷 흐름이 코드와 씬 데이터 기준으로 연결되어 있다.
- 향후 남은 과제는 실제 플레이 기준 동선과 밸런스 조정이다.
