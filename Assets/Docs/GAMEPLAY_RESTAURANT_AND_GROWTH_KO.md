# 종구의 식당 장사 및 성장 기능 문서

## 1. 허브 운영 개요

허브는 탐험 이후 정리와 성장, 장사를 처리하는 핵심 공간이다.
플레이어는 허브 안에서 직접 걸어 다니며 메뉴 선택, 창고 정리, 업그레이드, 지역 이동을 처리한다.

## 2. 장사 기능

### 메뉴 데이터

- 메뉴는 `RecipeData` 로 관리한다.
- 각 메뉴는 이름, 설명, 필요 재료, 판매가, 평판 변화량을 가진다.

### 장사 흐름

1. 메뉴 선택
2. 필요 재료 확인
3. 영업 실행
4. 결과 텍스트 확인
5. 골드 / 평판 반영

### 현재 확인 가능한 정보

- 선택 메뉴 설명
- 필요 재료
- 현재 보유량
- 조리 가능 여부
- 판매 결과
- 골드 획득량
- 평판 변화

## 3. 인벤토리와 창고

### 인벤토리

- 자원 전용 슬롯 구조
- 시작 8칸
- 확장 후 12칸
- 최종 16칸

### 창고

- 허브에서만 사용
- 맡기기
- 꺼내기
- 선택 품목 맡기기
- 선택 품목 꺼내기

창고는 탐험 후 남은 자원을 관리하고, 허브 루프를 더 부드럽게 연결한다.

## 4. 경제와 성장

### 골드와 평판

- 골드는 장사 결과와 업그레이드 비용에 사용된다.
- 평판은 숏컷 같은 편의성 해금에 연결된다.

### 업그레이드

현재 구현 중심은 아래 두 축이다.

- 인벤토리 확장
- 랜턴 해금

업그레이드는 골드와 특정 자원을 동시에 요구한다.

## 5. 허브 상호작용 지점

- 메뉴 선택대
- 영업대
- 창고
- 작업대
- 지역 포탈

각 지점은 `E` 입력 기준으로 동작하고, 조건이 맞지 않으면 막힌 이유를 텍스트로 안내한다.

## 6. 현재 구현 상태

- 메뉴 선택, 장사 실행, 결과 정산까지 이어진다.
- 창고는 선택형 입출고 흐름까지 구현되어 있다.
- 업그레이드는 현재 가능한 항목을 우선 실행하도록 정리되어 있다.
- 향후 남은 과제는 장사 수치와 업그레이드 비용 밸런스 조정이다.
