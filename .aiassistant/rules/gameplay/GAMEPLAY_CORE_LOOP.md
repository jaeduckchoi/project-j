# 코어 루프 기획

## 1. 하루 흐름

플레이어는 하루 단위의 탐험과 운영 사이클을 반복한다.

1. **허브 준비** : 골드, 평판, 인벤토리, 창고 상태를 확인하고 어디로 갈지 정한다.
2. **오전 탐험** : 지역에서 자원을 모으고 인벤토리 한도 안에서 무엇을 챙길지 판단한다.
3. **허브 복귀** : 탐험 종료 후 자동으로 오후 영업 단계에 진입한다.
4. **메뉴 선택** : 보유 재료에 맞는 메뉴 한 가지를 고른다.
5. **영업 진행** : 짧은 정산을 거쳐 골드와 평판이 변한다.
6. **정산** : 결과 텍스트를 확인하고 필요 시 업그레이드를 적용한다.
7. **다음 날** : 새로운 하루의 오전 탐험 단계로 돌아간다.

## 2. 단계 정의

- `Morning Explore`
  하루 시작 단계. 지역 포탈이 열려 있고 탐험 진입이 자유롭다.
- `Afternoon Service`
  허브 복귀 시 자동으로 진입한다. 메뉴 선택과 영업 단계.
- `Settlement`
  영업을 진행하거나 영업을 건너뛴 뒤 도달하는 단계. 다음 날 진행이 가능하다.

## 3. 의도하는 감각

- 오늘 어디로 갈지 정할 때의 기대감
- 인벤토리 안에 무엇을 담을지 결정하는 압박
- 깊이 들어갈지 돌아갈지 판단하는 탐험 긴장
- 모은 재료가 메뉴, 골드, 평판으로 변환되는 만족감
- 업그레이드가 다음 탐험을 여는 성장감

## 4. 단계 전이 규칙

- 탐험 지역 포탈은 `Morning Explore` 단계에서만 열린다.
- 허브 복귀 포탈은 오후 단계에서도 유지되어 하루 루프가 자연스럽게 끝나도록 한다.
- `Skip Exploration`, `Skip Service`, `Next Day` 행동은 현재 단계에 따라 노출과 잠금이 달라진다.
- 영업이 끝나면 결과 요약이 정산 단계의 안내 텍스트로 표시된다.
- 임시 가이드와 1회성 힌트는 단계 진입 시 한 번만 표시될 수 있다.

## 5. 단계별 플레이어 행동

### 허브 준비

- 현재 골드와 평판 확인
- 인벤토리/창고 정리
- 업그레이드 가능 여부 확인
- 방문할 지역 결정

### 오전 탐험

- 지역 전용 자원 수집
- 도구 요구 조건 확인
- 감속, 어둠, 돌풍 같은 환경 위험 체감
- 인벤토리 제한 안에서 수집 판단

### 복귀 후 영업

- 메뉴 선택
- 필요 재료 확인
- 영업 결과 계산
- 골드와 평판 획득

### 정산과 다음 날

- 결과 텍스트 확인
- 필요 시 업그레이드 적용
- 다음 날 진행

## 6. 시스템 연결 지점

- 단계 관리 : `DayCycleManager`
- 씬 이동 : `DayCycleManager.HandleSceneTravel`
- 단계 건너뛰기 : `SkipExploration`, `SkipService`, `AdvanceToNextDay`
- 영업 결과 → 정산 : `RestaurantManager` → `DayCycleManager.CompleteService`
- 임시 가이드 : `ShowTemporaryGuide`, `ShowHintOnce`
- 자동 회귀 점검 : `GameplayAutomationAudit` (단계 전이, 포탈 잠금, 팝업 일시정지)
